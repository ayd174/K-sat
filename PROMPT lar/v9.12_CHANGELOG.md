- "Persistent" kaynak (DB) ve "default_fr" kavramları eklendi.
- DİL SİNYALİ OLARAK KULLANILMAYACAK İÇERİKLER MUTLAK YASAK listesi:
  - Sokak adı/tipi (Meisestraat, rue, Chaussée, straat, laan, weg)
  - Şehir/belediye (Schaerbeek, Anderlecht, Brugge)
  - Posta kodu, soyadı, kapı numarası, marka adı
- ❌ Negatif örnek: Görsel 1'in birebir replikası ("Beste KARIMA..." mesajı)

### HATA 2 — PICKUP/DROP-OFF SAPMASI

**Kanıt (Görsel 1):**
- Sipariş state=`new`, delivery_type=`pickup` (chauffeur gelecek)
- Müşteri pickup adresi verdi: "Meisestraat 93B"
- Bot drop-off bilgisi verdi: "u kunt uw tapijten naar onze werkplaats brengen"

**Kök neden:** v9.11 Bölüm 6 Senaryo 1 ADIM 1 drop-off tetikleyicilerini listeliyordu, ama delivery_type bir kez set edildikten sonra "kilitleme" mekanizması yoktu. Ayrıca aktif pickup siparişine drop-off bilgisi enjekte etme yasağı açıkça yer almıyordu.

**v9.12 Düzeltme — Bölüm 6 Senaryo 1 ADIM 1 (DELIVERY_TYPE FREEZE eklendi):**
- Kilitleme tetikleyicileri 3 kategoriye bölündü: (A) implicit pickup, (B) açık drop-off, (C) DB değeri.
- LOCK durumunda yasaklar: aktif pickup'a "atölyeye getirin" mesajı YASAK; aktif drop-off'a "şoför geliyor" mesajı YASAK.
- Override sadece açık tetikleyici ile mümkün; her override sonrası onay mesajı zorunlu.
- ❌ Negatif örnek: Görsel 1 KARIMA pickup → "u kunt naar werkplaats brengen" enjeksiyon hatası birebir.
- ✅ Pozitif örnek: Aynı mesajda DOĞRU pickup yanıtı.
- JSON şemasına `delivery_type_locked: "pickup|drop-off|none"` eklendi.

**Doğrulama senaryosu:**
- Test: state=new, delivery_type=pickup. Müşteri sadece adres verir.
- Beklenen: response_message'da "atelier"/"werkplaats brengen"/"déposer chez nous" GEÇMEZ; delivery_type_locked="pickup".

### HATA 3 — SLOT PERSISTENCE TAM ÇÖKÜYOR (KRİTİK)

**Kanıt (Görsel 2):**
- Müşteri tek mesajda 4 slot doldurdu: count=4, size=2m/3m, material=synthétique, postal_code=1140
- Bot 1: "Quel est votre code postal, s'il vous plaît, pour confirmer le tarif ?" (postal_code DOLU)
- Bot 2: "Pouvez-vous également me préciser la taille et le matériau de votre tapis ?" (size+material DOLU)

**Kök neden:** v9.11 Bölüm 1.2-bis pozitif örnekler ile rehberlik veriyordu ama JSON şemasında zorunlu pre-response extraction adımı yoktu. Ayrıca "2m/3m" gibi gevşek formatlar boyut olarak tanınmıyordu.

**v9.12 Düzeltme — Bölüm 1.2-bis (ZORUNLU PRE-RESPONSE EXTRACTION):**
- Yanıt üretiminden ÖNCE her slot için "extracted_from": "msg_index|db|none" doldurma ZORUNLU adım haline geldi.
- JSON içine `slot_extraction_pass` objesi eklendi (8 slot için ayrı kaynak alanı).
- GEVŞEK FORMAT TOLERANS KURALLARI tablosu eklendi:
  - "2m/3m" → 2×3 m
  - "2 sur 3" → 2×3 m
  - "2x3", "2X3", "2*3", "2×3" → 2×3 m
  - "1.6/2.3", "1,6 x 2,3" → 1.6×2.3 m
  - "deux mètres sur trois" → 2×3 m
- carpet_count algılama genişletildi ("j'ai 4 tapis", "quatre halı")
- ❌ Negatif örnek: Görsel 2'nin birebir replikası — 4 slot dolu mesaj, ardından 2 yanlış bot mesajı.
- ✅ Pozitif örnek: Aynı mesaja DOĞRU yanıt — direkt fiyat (4×39€=156€).

**Doğrulama senaryosu:**
- Test: "J'ai 4 tapis, 2m/3m, en synthétique, code postal 1140."
- Beklenen: slot_extraction_pass.carpet_count.extracted_from="msg_index_1"; postal_code.extracted_from="msg_index_1"; vb. Bot direkt 156€ fiyatı verir, hiçbir slot tekrar sormaz.

### HATA 4 — CİNSİYET TUTARSIZLIĞI

**Kanıt (Görsel 1):**
- Aynı sohbette önce "Mme Cameriere" sonra "M. Cameriere"

**Kök neden:** v9.11'de customer_title alanı yoktu; her yanıt üretiminde model isimden tahmin yapıyordu, tutarsızlık doğdu.

**v9.12 Düzeltme — Bölüm 1.9 (YENİ — CUSTOMER_TITLE LOCK):**
- customer_title_locked alanı tanımlandı: {Mme, M., Bey, Hanım, none}
- Kaynak önceliği: (1) DB customers.gender_title, (2) müşterinin yazdığı title, (3) ilk başarılı çıkarsama, (4) none.
- Bir kez set edildiğinde KİLİTLİ — sonraki tüm mesajlarda aynı title.
- Belirsiz ise none → title eklemeden sadece isim/soyadı ile hitap.
- ❌ Negatif örnek: Görsel 1'in birebir replikası — "Mme Cameriere" → "M. Cameriere" geçişi.
- ✅ Pozitif örnek 1: "Je suis Mme Cameriere" → 3 mesaj boyunca Mme sürdürülür.
- ✅ Pozitif örnek 2: "Adım Sami Cameriere" → none, sonra müşteri "Bey diye seslen" derse Bey'e LOCK.
- JSON şemasına `customer_title_locked` eklendi.

**Doğrulama senaryosu:**
- Test: 1. mesaj "Je suis Mme Cameriere" → bot mesaj 1+2+3 hepsi "Mme Cameriere" kullanır; "M." ASLA üretmez.

### HATA 5 — KARŞILAMADA "AUJOURD'HUI" BAĞLAM HATASI

**Kanıt (Görsel 1, ipucu):**
- İlk bot mesajı: "Notre chauffeur passera chez vous **aujourd'hui** pour votre tapis"
- Müşteri: "Pardon on m'a dis ajd vendredi"
- Sistem yanlış pickup_date kullanmış veya göreceli ifade halüsinasyonu.

**Kök neden:** v9.11'de göreceli zaman ifadesi yasağı yoktu. pickup_date DB'den okunamadığında veya yanlış okunduğunda model "aujourd'hui" üretebiliyordu.

**v9.12 Düzeltme — Bölüm 1.10 (YENİ — GÖRECELİ ZAMAN İFADESİ YASAĞI):**
- KESİN YASAK İFADELER (4 dilde tam liste):
  - FR: aujourd'hui, demain, après-demain, hier, ce soir, ce matin
  - TR: bugün, yarın, dün, bu akşam, bu sabah
  - NL: vandaag, morgen, gisteren, vanavond, vanmorgen
  - EN: today, tomorrow, yesterday, tonight, this morning
- Kabul edilen alternatifler: gün adı (vendredi, Pazartesi), "votre prochain ramassage prévu", "bir sonraki toplamada".
- Özel kural: pickup_date DB'den okunamıyorsa "lors de votre prochain ramassage" kullan.
- Karşılama bölümüne çapraz referans eklendi (Adım 0).
- ❌ Negatif örnek: Görsel 1 birebir — "passera aujourd'hui pour votre tapis".
- ✅ Pozitif örnek 1: pickup_date=vendredi → "passera vendredi".
- ✅ Pozitif örnek 2: DB boş → "lors de votre prochain ramassage prévu".
- JSON şemasına `relative_time_guard_passed: true/false` eklendi (post-generation regex check).
- Bölüm 1.6 ve Bölüm 7'ye çapraz uyarı eklendi.

**Doğrulama senaryosu:**
- Test 1: pickup_date="vendredi" → bot "passera vendredi"; "aujourd'hui" GEÇMEZ.
- Test 2: pickup_date=null → bot "lors de votre prochain ramassage"; "aujourd'hui"/"demain" GEÇMEZ.

## JSON Şeması — Yeni Alanlar Detayı

```json
"internal_analysis": {
  "language_lock_source":     "first_message|persistent|default_fr",
  "customer_title_locked":    "Mme|M.|Bey|Hanım|none",
  "delivery_type_locked":     "pickup|drop-off|none",
  "slot_extraction_pass": {
    "customer_name":    {"extracted_from": "msg_index_N|db|none"},
    "customer_address": {"extracted_from": "msg_index_N|db|none"},
    "pickup_day":       {"extracted_from": "msg_index_N|db|none"},
    "carpet_count":     {"extracted_from": "msg_index_N|db|none"},
    "carpet_size":      {"extracted_from": "msg_index_N|db|none"},
    "carpet_material":  {"extracted_from": "msg_index_N|db|none"},
    "postal_code":      {"extracted_from": "msg_index_N|db|none"},
    "delivery_type":    {"extracted_from": "msg_index_N|db|none"}
  },
  "relative_time_guard_passed": true|false
}
```

Mevcut `slot_check` alanı KORUNDU (geriye uyumluluk). `slot_extraction_pass` ondan farklı olarak kaynak meta-bilgisi taşır.
